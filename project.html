DevOps CI/CD Pipeline 구축

2024.12 ~ 진행중

DevOps란 개발팀과 운영팀이 협업하여 빠른 배포 주기를 자동화하는 프로세스, 문화, 방법을 말합니다. DevOps를 구현하기 위한 방법으로는 지속적인 통합(CI)와 지속적인 배포(CD)를 의미하는 CI/CD 파이프라인을 이용한 프로젝트를 진행하였습니다.




Pipelein 아키텍처

파이프라인은 AWS의 서비스를 이용하여 오른쪽과 같이 구성하였습니다.
  1. GitHub에서 Code가 Push되면 파이프라인이 트리거
  2. Docker 이미지 생성 및 SCA, SAST 분석 후 S3 버킷에 업로드
  3. 빌드된 Docker 이미지를 ECR 프라이빗 리포지토리에 저장
  4. 지정된 Cluster에 Task 정의를 통해 서비스를 Rolling Update 배포
  5. OWASP ZAP으로 ECS 서비스 스캔 후 S3 버킷에 업로드
  6. CloudWatch의 로그 및 메트릭 수집 내용을 Grafana로 시각화
